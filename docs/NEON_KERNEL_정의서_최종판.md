## 0. 정정 요약 (이전 정의서 대비 무엇이 바뀌었나)

| 항목 | 이전 정의서(오류) | 정정판(원결정) |
|---|---|---|
| 조작 기본 상태 | 무입력 = 자동 연사(정지) | **무입력 = 슬롯·조준선 자동 회전 + 자동 연사** |
| 꾹(홀드) | 회전(사격 없음) | **브레이크(정지) + 6방향 고정 방어 격자 사격 + 체력 소모** |
| 탁(짧게) | 손 떼기 = 발사 | **회전 방향 반전(시계 ↔ 반시계)** |
| 보상 획득 | 순차 점멸 + Tap 선택 + 슬롯 회전 후 배치 | **정비 궤도 3선택 → 사격선 역흡수로 슬롯 인입** |
| 코어/태그/기체 시스템 | 대부분 누락 | **전체 복원** |

**이번 세션에서 추가 확정된 주요 사항(상세는 각 장):**
- **합성:** 겹쳐 압축 → **인접 슬롯 자동 압축**(3.3). 기체 고유 기능: 압축 폭발/클린 레이저 폐기 → **오버히트 노바**(7장).
- **보상:** 전투 중 코어 드롭 삭제 → **정비 궤도 일원화** + 전투 버프 토큰 4종(3장). 하이브리드 촉매 폐기 → **슈퍼 정비 궤도**(3.7).
- **기체:** 글리치-Zero = **과부하 누커**(8.1), 딥웹 = **와일드 압축 잭팟 + 제로데이**(8.2)로 전면 재설계.
- **아웃게임:** 사망/환전 루프·단일 통화·슬롯 진척 조건·병렬 3갈래 성장(9.3~9.5).
- **적:** 이산 "방향 수" 폐기 → **연속 각도 공습 모델 + 예산 기반 랜덤 스폰**(10.3~10.5) + 3섹터 강도 곡선(11.2).
- **체력:** 0~100 연속값·피해 3종·0.5초 무적(2.3).

## 1. 핵심 철학

### 1.1 컨셉
플레이어는 끝없는 **디지털 싱크홀**로 침몰하는 고철 데이터 처리선(마스터 커널)이다. 사방에서 밀려오는 것은 괴물이 아니라 버려진 악성 코드·정크 데이터·깨진 픽셀 파편이다. 슬롯에 장착하는 코어들은 "버려진 현대인의 디지털 잔해"이며, 용량 최적화를 위해 압축(Zip)한다는 설정이 곧 게임 시스템 그 자체다.

### 1.2 원버튼 미니멀리즘
- 기체는 화면 정중앙에 **완벽히 고정**되며 절대 이동하지 않는다.
- 모든 조작은 화면 특정 위치가 아닌 **화면 전역 단일 터치**로만 이루어진다.

### 1.3 텍스트 제로 (Zero-Text UI)
- HP 바, 대미지 숫자 폰트, 스탯 텍스트, 인벤토리 창이 존재하지 않는다.
- 모든 정보는 **홀로그램의 기하학적 형태 변화 / 네온 컬러 / 글리치(노이즈) 이펙트**로만 전달한다.

## 2. 조작 체계 (최종 확정 = 2안)

게임 내 모든 상호작용(전투·보상·코어 장착)은 아래 단일 규칙을 벗어나지 않는다.

### 2.1 인게임 전투
- **기본 상태 (무입력):** 기체 주변 6궤도 슬롯과 각 슬롯의 사격 조준선이 **시계방향으로 자동 회전**한다. 각 슬롯의 무기는 자신이 향한 방향으로 **자동 연사**한다. (회전하는 다연장 포탑 상태)
- **탁 (짧게 누름):** 누를 때마다 회전 방향이 **즉시 반전**된다 (시계 ↔ 반시계). 사방에서 조여오는 적에 맞춰 조준 라인을 반대편으로 빠르게 트는 핵심 컨트롤.
- **꾹 (길게 누름 / 홀드):** 회전이 **브레이크(정지)**되어, 6개 코어가 **각자 바라보던 방향 그대로 고정 일점사**한다(한 점에 모으는 게 아니라, 멈춘 순간의 6방향 배치가 유지됨). 단, 홀드 중에는 엔진 과열로 **초당 체력이 소량 소모**된다(리스크).

> 집중사격은 12시 고정이 아니라 **각 코어가 멈춘 순간의 자기 방향**으로 일점사한다.

#### 브레이크 = 집중 방어 격자 (체력 비용의 진짜 대가)
브레이크는 공격 강화이자 **확실한 방어수단**이다. 평상시 회전사격은 6방향이 계속 돌아 어느 방향도 잠깐씩만 커버되지만(스쳐간 틈으로 적이 파고듦), 브레이크는 그 순간의 6방향 배치를 **고정 방어 격자**로 만든다.

- 각 코어가 자기 정면을 지속적으로 틀어막아 그 라인으로 들어오는 적·투사체를 확실히 **요격**.
- **투사체 상쇄:** 브레이크 사격이 적 투사체(비트 슈터 탄 등)를 맞받아 지운다(평상시 회전 중엔 놓칠 수 있음).
- **판단의 핵심 = 멈추는 각도와 타이밍:** 적 분포에 맞춰 6방향 격자가 딱 들어맞는 순간 멈추면 사방을 효율적으로 막고, 어설피 멈추면 코어 사이 빈 각도로 샌다. 탁(반전)으로 조절하다 최적 순간에 꾹으로 고정.
- **올인의 리스크:** 회전이 멈추니 코어가 없는 각도(슬롯 미개방·빈 슬롯)는 무방비. "지금 제일 위험한 쪽을 피 깎아 막을까 vs 계속 돌며 얕게 훑을까"의 판단이 체력 비용으로 정당화된다.
- 슬롯을 다 채울수록 방어 격자가 촘촘해진다 → 슬롯 성장과 직결.

체력 소모는 이 "6방향 동시 고정 방어 + 집중 화력 + 노바 충전(7장)"을 동시에 사는 비용이다.

#### 브레이크 체력 소모 (과열 단계)
연속 홀드 시간에 따라 드레인이 가속된다. 짧고 정밀한 점사·코어 흡수는 거의 무료, 길게 버티며 극딜하는 건 비싸게 설계한다. **0.5초 이상 손을 떼면 단계 리셋.**

| 연속 홀드 시간 | 초당 체력 소모 |
|---|---|
| 0 ~ 1.5초 (워밍업) | 1% / 초 |
| 1.5 ~ 3초 (과열) | 3% / 초 |
| 3초 이상 (과부하) | 5% / 초 |

- 0.3~0.5초짜리 점사/흡수 브레이크는 실질 0.5% 이하 → 사실상 무료.
- 탱키한 적 3초 일점사 ≈ 6% 소모(느껴지지만 회복 가능).
- 드레인이 셀수록 기체 중심 배터리가 더 붉고 거칠게 떨려, 숫자 없이 위험도가 읽힌다.

#### 체력 회복 장치
- 웨이브 클리어(정비 궤도 진입) 시 **+10% 회복**.
- 적 처치 시 낮은 확률로 소량 회복.
- 정비 궤도에서는 드레인이 없다 → 브레이크 리스크는 "웨이브 안에서 관리하는 자원".

#### 기체별 변형
- **글리치-Zero:** 글리치 로그 스택당 드레인 −15%, 최대 6스택에서 **드레인 0**.
- **프록시:** 드레인 대신 **초당 +1% 보호막 충전**(탱커 정체성). 브레이크가 곧 방어 행동이 된다.

### 2.2 컨트롤 손맛 요약
탁·탁으로 회전 방향을 튕겨 적이 몰린 쪽으로 사격 라인을 정렬 → 꾹으로 그 각도에 일점사. 위기엔 탁 연타로 양방향을 번갈아 막는다.

### 2.3 체력 · 피격 규칙
중앙 고정 게임이라 "회피"가 없다. **요격(다가오는 적·탄을 사격으로 지우는 것)이 곧 방어**이고, 체력은 "제때 못 막은 것에 대한 누적 벌점"이다.

- **체력 표현:** 내부적으로 **0~100 연속값**을 갖되 숫자로 보이지 않는다. 시각은 3구간 매핑 — 정상(70%+, 깔끔한 네온) / 경고(35~70%, 글리치 노이즈) / 위험(35%-, 붉은 테두리 비네팅·이명·파편 탈락). 브레이크 드레인과 피격이 **같은 체력 풀**을 공유.
- **피해 원인 3종(구체 수치는 밸런싱 패스):**
  - **적 본체 충돌(중앙 도달):** 요격 못한 돌격형이 닿음 → 가장 큰 피해, 적 종류·티어별 차등(잡몹 5~10%, 그레이 탱커 더 큼).
  - **적 투사체 피격:** 비트 슈터 단발탄 등 → 중간(3~5%).
  - **환경/기믹:** 스팸 마인 폭발, 보스 패턴 → 큼(10%+).
- **적 중앙 도달 처리:** **충돌 시 1회 피해 후 적 소멸**(요격 실패 = 한 대 맞고 끝). 달라붙어 지속 피해 주거나 즉사시키지 않는다.
- **무적 시간:** 피격 직후 **0.5초 i-frame**(기체 깜빡임). 사방 동시 피격 즉사 방지. 단 환경 지속 피해·보스 장판처럼 머무는 위험은 무적 무시/단축.
- **최대 체력·회복:** 최대치 100 기준. 회복은 웨이브 클리어 +10% / 적 처치 낮은 확률 소량 / 버프 토큰(냉각 등). 최대 체력 자체는 아웃게임 피지컬(9.4 ②)로 영구 증가 → "초반엔 몇 대만 맞아도 죽지만 성장하면 탱킹 여유".
- **브레이크와의 상호작용:** 오래 브레이크 → 드레인으로 체력↓ + 그동안 무방비였던 각도 피격 → 급사 위험(의도된 긴장). 단 오버히트 노바(7장)가 이 위험의 보상 출구.

## 3. 코어 획득 및 크래프트 (사격선 역흡수)

### 3.0 시작 조건 — 기본 시작 코어 (필수 전제)
빈 슬롯은 사격을 못 하므로(코어 = 무기), 6칸이 전부 비어 시작하면 첫 정비 궤도 전까지 무기가 0개가 된다. 따라서 **게임 시작 시 티어1 코어 1개를 1번 슬롯에 기본 장착**한 채 출발한다(편의가 아닌 불변 기본값).

- **시작 코어 풀:** 단독 성능이 안정적인 **순수 직선형 4종** 내에서 무작위 — 스트림 핑(패킷) / 도트 스니펫(소스) / 풀 인덱스(마이그레이션) / 헤비 정크(스크랩). 변칙·산탄형은 시작에서 제외해, 어떤 시작이든 1웨이브를 무난히 넘길 하한을 보장.
- **최초 1판 예외:** 신규 유저의 **첫 플레이 1판은 스트림 핑(가장 직관적) 고정**(사격 기본기 학습). 이후 4종 랜덤 해금.
- **가중 시드:** 랜덤으로 정해진 시작 코어의 색이 3.6 주력 색 가중의 **출발 시드**가 된다(약하게만 — 다른 방향으로 틀 자유는 유지).
- **아웃게임 부스트와의 관계:** "시작 코어 보유" 부스트는 기본 1개에 더해 **+1(무작위 저티어)**을 얹는 것으로 재정의(9장).

### 3.1 정비 궤도 (쉬어가는 구간) — 코어 획득의 유일한 경로
웨이브 클리어 시 전투가 잦아들고 **안정 도킹 궤도**로 전환된다. 적 스폰·낙사 기믹이 일시 정지하므로 손가락을 쉬며 빌드를 설계할 수 있다(이때는 브레이크 체력 소모 없음). 기체 주변에 **3개의 후보**가 공전한다.

- **코어(영구 빌드 자원) 획득은 오직 정비 궤도에서만** 일어난다. 전투 중에는 코어가 드롭되지 않는다(전투는 순수 생존·사격에 집중).
- **발동 시점:** 매 웨이브 클리어 시마다(30웨이브 = 최대 30회).
- 일반 웨이브 → 3선택 정비 궤도.
- 중간 보스(10, 20) → **슈퍼 정비 궤도**(아래 3.5).
- 최종 보스(30) → 게임 종료, 보상 없음.

### 3.2 사격선 역흡수 (조작법 2.1과 100% 동일)
- 평소처럼 슬롯·사격선이 자동 회전한다. **탁**으로 회전 방향을 반전해 미세 조정한다.
- 내가 강화/융합하려는 **슬롯의 사격선**을 원하는 후보 코어에 정렬한 뒤 **꾹(브레이크)**으로 멈춘다.
- 멈춘 사격선(레이저) 위에 얹힌 코어가 **그 선을 타고 역으로 빨려 들어와** 해당 슬롯에 장착된다.
- **한 번의 브레이크당 코어 1개만** 흡수된다(오조작 방지). 흡수 전에 손을 떼면 코어는 관성으로 튕겨 나간다(거부).

> 사격선 역흡수는 **정비 궤도 전용** 메커니즘이다(전투 중 코어 드롭 없음).

### 3.3 장착 결과 — 인접 슬롯 자동 압축
코어는 **항상 빈 슬롯에만** 들어간다(기존 코어 위에 겹쳐 쓰지 않음). 들어온 직후 **양옆 인접 슬롯**을 검사해 압축을 판정한다. 압축은 **기체 고유 기능과 무관**(압축 폭발 폐기 — 7장).

#### 압축 성립 조건 (티어별)
- **T1 + T1 → 단색 T2:** **같은 색일 때만** 압축. (각 색 T1은 2종이므로 색당 3조합 = A+A · B+B · A+B → 6.2의 동색 T2 12종과 1:1 대응.)
- **T2 + T2 → T3:** **색 무관** 압축. 단색 T2 두 개의 색 조합 = 4색 중 2개 = 동색 4 + 이종 6 = **정확히 10조합**으로 6.3의 10대 색조합 지정효과와 1:1 대응.
- **T3 = 천장:** T3 + T3는 압축되지 않는다(이후 강화는 스탯 오버클록·Affix 영역).
- **인접에 위 조건을 만족하는 짝이 없으면:** 그 슬롯에 독립 장착(압축 없음).
- **압축 여부는 "어느 빈 슬롯에 꽂느냐"로 100% 통제된다.** 짝 옆에 꽂으면 압축, 멀리 꽂으면 독립 유지.

#### 베이스·결과물 위치 (= 플레이어의 선택 레버)
두 코어가 압축될 때, **둘 중 반시계 쪽 코어가 베이스**(공격 형태 유지)가 되고 **그 자리에 상위 티어 결과물이 생성**된다. 시계 쪽 코어는 재료(스탯 기증)가 되고 그 슬롯은 빈칸이 된다.
- 따라서 플레이어는 새 코어를 기존 코어의 **시계 쪽**에 꽂으면 기존 코어가 베이스, **반시계 쪽**에 꽂으면 새 코어가 베이스가 되도록 **베이스를 직접 고른다.** ("어느 칸에 꽂느냐 = 압축 여부 + 베이스"까지 통제.)
- **충돌 우선순위:** 흡수한 코어가 양옆 모두 압축 가능하면 **흡수한 코어 기준 반시계 방향 이웃을 우선** 압축한다.
- **방향 무관 불변식:** 어느 쪽으로 꽂든 합쳐지는 두 코어의 **색 조합은 동일**하므로, T3의 색조합 특수효과(6.3 10종)는 방향과 무관하게 같다. 방향으로 달라지는 것은 **베이스 형태와 스탯 합산 결과**뿐이다.

#### 연쇄 압축 (T1+T1 → T2 → 곧바로 옆 T2와 T3)
압축으로 새 코어가 생기면, **그 결과 코어를 기준으로 인접 검사를 다시 돌린다**(반복). 이로써 한 번의 흡수 안에서 T1 두 개가 T2가 되고, 그 T2가 옆 T2와 다시 압축돼 T3까지 연달아 만들어질 수 있다.
- **방향 특성(자연 발생):** 위 베이스 규칙상, 기존 T2의 **시계 쪽**에서 만들어진 새 T2만 자연히 기존 T2와 인접해 연쇄된다. (반시계 쪽은 기존 T2가 슬롯을 점유해, 새 T2가 기존 T2 바로 옆 자리에 생길 수 없음.)
- **반시계 예외처리 (연쇄 우선):** 어떤 T1+T1 압축의 결과물을 **기본(반시계) 자리에 두면 추가 압축이 불가하지만, 시계 자리에 두면 즉시 상위 압축이 가능해질 경우, 자동으로 결과물을 시계 자리로 옮긴다**(= 베이스를 시계 쪽 코어로 변경). 이 경우 T3의 베이스는 새로 만든 T2가 된다.
  - **탐색 깊이 = 1수 앞보기:** 예외는 "옮기면 *바로 다음 한 번*의 압축이 가능"할 때만 발동(임의 깊이 재배치 금지 — 예측 가능성·플레이어 통제권 보존).
  - **양방향 동시 가능 시:** 기본(반시계) 자리로도 연쇄가 성립하면 예외를 발동하지 않고 기본을 유지(반시계 우선, 위 충돌 우선순위와 일관).
- **프리뷰 반영(3.4):** 고스트 홀로그램은 예외로 뒤집힌 경우까지 포함해 **최종 T3의 위치·베이스**를 미리 보여준다.

> 📝 **[AI 구현 — 압축 해소 알고리즘]** 한 번의 흡수/덮어쓰기 후 아래를 순서대로 1패스 실행한다.
> 1. 새 코어를 대상 슬롯에 배치(만석이면 덮어쓰기 흡수 — 아래).
> 2. `resolve(slot)` 호출: 해당 코어의 양옆 인접 슬롯에서 "성립 조건"을 만족하는 짝을 찾는다.
>    - 양옆 모두 가능 → **반시계 이웃 우선** 선택.
>    - 짝 없음 → 종료(독립 장착).
> 3. 압축 실행: 짝과 새 코어 중 **반시계 쪽 = 베이스**, 그 자리에 결과 코어 생성, 시계 쪽 슬롯 비움.
>    - **예외 검사(T1+T1 한정):** 기본 배치(결과=반시계 자리)로는 다음 압축이 불가하나, 결과를 시계 자리에 두면 즉시 T2+T2가 성립한다면, 결과 위치를 시계 자리로 뒤집는다(베이스=시계 쪽 코어). 단 1수만 앞본다.
> 4. 결과 코어 슬롯을 인자로 `resolve()`를 **재귀 호출**(연쇄). T3에 도달하면 더는 압축 불가하므로 자연 종료.
> 5. 매 단계 결과는 3.4 고스트 프리뷰로 동일 알고리즘을 미리 1회 돌려 표시(실제 적용 전).

#### 슬롯 만석 시 (빈칸 없음) — 덮어쓰기 흡수
6칸이 다 찬 상태에서 새 코어를 흡수하면, 사격선을 정렬해 꾹으로 멈춘 **그 슬롯의 기존 코어가 파괴되고 새 코어가 즉시 그 자리에 들어온다.** 별도 정리 단계 없이 흡수 동작 하나로 파괴+장착이 합쳐진다 → "어느 칸에 꽂느냐 = 어느 코어를 버리느냐". (망한 빌드를 갈아엎어 복구할 기회 보장.)

> 덮어쓰기로 들어온 새 코어도 한 박자 뒤 양옆 인접 압축 판정을 받는다.

### 3.4 직관적 인지 (텍스트 없이)
- 후보 코어는 **속성 컬러**(아래 4대 태그)로 0.1초 만에 종류를 알린다.
- 코어를 슬롯에 정렬하면, 슬롯 위에 **합성 결과물의 고스트 홀로그램**과 **스탯 변동 프리뷰**(예: 사격속도 잔상이 빨라짐 / 기체 중심 배터리가 붉게 과열)가 미리 뜬다.

### 3.5 보상 풀 구성 (3선택지)
정비 궤도의 3개 후보는 섹터(성장 단계)에 따라 두 유형이 비율을 달리해 등장한다.

- **유형 A — 완제품 티어1 코어:** 빌드의 재료. 색상·하위 분기(스트림/스캐터 등)가 정해진 완제품.
- **유형 B — 스탯 오버클록:** 새 코어 대신 이미 장착한 무기의 6대 스탯 중 하나를 강화. 빌드가 막혔거나 현재 무기를 더 키울 때.

| 섹터 | A(완제품 코어) | B(스탯 오버클록) |
|---|---|---|
| 1 (1~9) | 80% | 20% |
| 2 (11~19) | 55% | 45% |
| 3 (21~29) | 40% | 60% |

> 초반은 코어를 모아 빌드를 짜고(A 위주), 후반은 완성된 무기를 스탯으로 극한 펌핑(B 위주)하는 흐름. **하이브리드 촉매(티어3 확정 지급)는 폐기** — 빌드 설계권을 유저에게 남긴다.

### 3.6 억까 최소화 로직 (가중 추첨 3규칙)
완전 무작위가 아니라, 정비 궤도가 뜰 때 1회 도는 가벼운 가중 추첨. 별도 패스 기능은 두지 않는다(이 로직만으로 충분). 가끔의 운 꼬임은 "이번 판은 이상하네, 다음 판 하지" 수준으로 허용.

1. **주력 색 가중:** 현재 6슬롯에서 가장 많이 쓰는 색상의 코어가 후보 풀에서 뽑힐 확률을 상향.
2. **다양성 강제:** 3개 후보가 전부 같은 유형/색이 되지 않도록 결이 다른 선택지를 최소 1개 섞음.
3. **연속 실패 보정(피티):** 직전 궤도에서 아무것도 안 골랐거나 빌드 부적합 코어를 먹었으면, 다음 궤도에서 주력 색 가중을 한 번 더 상향.
4. **빈 슬롯 코어 보장:** 빈 슬롯이 1칸 이상 있으면 3후보 중 **최소 1개는 반드시 코어(A)**로 구성(빈칸인데 스탯강화만 떠서 성장이 막히는 상황 방지).

### 3.7 슈퍼 정비 궤도 (중간 보스 보상 — 안1+2 결합)
중간 보스(10, 20) 클리어 시 일반 3선택 대신 발동.

- 후보가 **5~6개**로 펼쳐지고, 그중 **2개를 연속으로** 가져간다(코어를 평소보다 많이, 내가 골라서 확보 → 빌드업 가속).
- 이 궤도는 **슬롯 회전이 느려져** 원하는 슬롯에 정밀하게 꽂을 수 있다.
- 무엇을·어디에 넣을지는 100% 유저 결정 → 티어3은 확정 지급이 아니라 "선택지가 풍부해진 결과"로 자연스럽게 완성된다.

### 3.8 전투 중 버프 토큰 (일시 버프, 빌드 불변)
적 처치 시 낮은 확률로 그 자리에 **버프 토큰**이 잠시 떠 있고, 회전하는 사격선이 스치면 일정 시간(8~10초) 자동 발동한다. 흡수를 위해 멈출 필요 없음 → 전투 흐름을 끊지 않는다.

- 영구적이지 않고 빌드를 바꾸지 않는다(오염 없음). 4대 속성과 헷갈리지 않게 **흰색·골드 계열**로 통일.
- **오버드라이브:** 전 무기 발사속도 급증.
- **실드:** 피격 1~2회 무효.
- **냉각:** 브레이크 체력 소모 일시 0(마음껏 일점사).
- **정화:** 화면 전체 잡몹 약화/소멸.

## 6. 코어 티어 구조

### 6.1 티어 1 — 기초 코어 (8종)
속성별 2종. 같은 속성 안에서도 메커니즘이 갈리는 하위 분기.

- 패킷: **스트림 핑**(직선 연사), **스캐터 핑**(근거리 산탄)
- 소스: **도트 스니펫**(단일 과열 DOT), **에어리어 스니펫**(소형 장판)
- 마이그레이션: **풀 인덱스**(인력 탄), **딜레이 인덱스**(허공 정지 후 가속 변칙)
- 스크랩: **헤비 정크**(중량 넉백), **플래시 정크**(충돌 시 파편 분열)

### 6.2 티어 2 — 동색 결합 (12종)
같은 색상 코어 2개가 **인접 슬롯에 모일 때** 자동 압축되어 진화(3.3). 각 속성당 **순혈 2종 + 동색 융합 1종**.

| 속성 | 순혈 A | 순혈 B | 동색 융합(A+B) |
|---|---|---|---|

### 6.3 티어 3 — 컴포넌트 조립형 (조합식, 무한 변수)
별도 무기를 하드코딩하지 않는다. **티어2 코어 2개**가 인접 슬롯에서 자동 압축될 때(색 무관, 3.3):

> ✅ **[확정 — 압축 규칙 닫힘]** **T1+T1은 같은 색일 때만 → 단색 T2 / T2+T2는 색 무관 → T3.** 단색 T2 두 개의 색 조합 = 4색 중 2개 = 동색 4 + 이종 6 = **정확히 10조합**으로 아래 "10대 색조합 지정효과"와 1:1 대응한다(8종 T1 → 12종 T2 → 10조합 T3로 6.1/6.2/6.3 수치가 모두 맞물림). 베이스·결과 위치·연쇄·반시계 예외는 **3.3** 참조.

1. **형태 유지:** 베이스(압축 시 반시계 쪽 T2) 무기의 투사체 형태·궤적·사운드를 그대로 계승.
2. **스탯 합산:** 재료 코어의 6대 스탯(연사·투사체수·크기·지속 등)을 엔진에서 `최종값 = 베이스 + (재료 × α)`로 합산. **α는 둘로 분리한다:** 동색 압축용 `α_동색`(특수효과 없이 순수 체급 폭발 → 높게)과 이종 압축용 `α_이종`(특수능력을 얹으므로 → 낮게). 구체값은 밸런싱 패스(15장). *(엔진은 두 α 슬롯을 데이터로 보유만 하고, 수치는 후입력.)*
3. **색조합 지정효과 주입:** 두 코어의 **색상 조합(총 10가지)**에 따라 사전 지정된 특수효과가 투사체에 부여됨. 색 조합은 압축 방향과 무관하므로 **특수효과는 방향 불변**, 베이스 형태·스탯만 방향에 따라 달라진다(3.3 불변식).

이 3중 레이어로 개발 에셋은 28개 수준으로 통제하되 유저 체감 조합은 수천 가지가 된다.

#### 10대 색조합 지정효과
**같은 색(4종) = 순혈 오버클록(체급 폭발, 메커니즘 유지):**
- 민트×민트 = **하이퍼 클록** (탄속·회전속도 대폭 증가)
- 레드×레드 = **멜트다운** (전탄 치명타, 과열 지속 3배)
- 퍼플×퍼플 = **이벤트 호라이즌** (판정 200%↑, 적 투사체 흡수→대미지)
- 그레이×그레이 = **타이탄 웨이트** (투사체 1↓, 단발 깡댐 400%↑, 보스 경직)

**다른 색(6종) = 이종 합선(신규 메커니즘):**
- 민트×레드 = **코드 과부하** (연사 2배 + 과열 스택 100% 시 폭발)
- 민트×퍼플 = **동적 동기화** (투사체끼리 중력 레이저 실로 연결, 닿는 적 홀딩)
- 민트×그레이 = **버퍼 분열** (적중/소멸 시 베이스 소형 복사본 3개 유도 분출)
- 레드×퍼플 = **디아블로 코어** (명중 지점에 미세 인력 장판 + 과열)
- 레드×그레이 = **열역학 파쇄** (장갑 100% 파괴, 방어력 0 → 받는 피해 1.5배)
- 퍼플×그레이 = **마그네틱 임팩트** (넉백 300%↑, 벽·적 충돌 시 광역 스플래시)

### 6.4 접두/접미 모듈 (Affix) — 엔드 콘텐츠
- **접두(형태) 10종**(예: 분열의, 유도의, 거대한, 궤도의), **접미(효과) 10종**(예: 과부하, 치명적인, 가속의, 포맷팅).
- 드롭 규칙: 일반 70% / 단일 모듈 25% / **글리치 에러 코어(접두+접미 동시) 5%**.

## 7. 기체 고유 기능 — 오버히트 노바 (Overheat Nova)

> ⚠️ 기존 "커널 오버플로우 폭발 / 슬롯 클린 레이저"는 **폐기.** 합성이 전투 중 겹쳐 압축이 아니라 정비 궤도의 인접 자동 압축으로 바뀌어, 압축을 트리거로 한 위기탈출기가 성립하지 않기 때문. 대신 **브레이크 과열**을 트리거로 재설계한다.

브레이크(꾹)를 길게 유지해 **과부하 단계(3초 이상, 초당 5% 드레인 구간)**에 머무는 동안 기체 중심에 **과열 게이지**가 차오른다.

- 게이지가 가득 차면 기체가 임계에 도달한다.
- 이때 **브레이크를 떼는 순간**, 축적된 열이 360도 글리치 충격파로 방출된다 → 접근한 잡몹 소멸 + 정예 경직.
- "피를 깎으며 오래 일점사 → 임계 도달 → 떼면서 화면 정화"라는 고위험 고보상 출구. 장시간 브레이크의 체력 리스크에 보상을 달아준다.

#### 기체별 결
- **글리치-Zero:** 드레인 0 + 사격 과열이 노바 게이지로 전환 + 노바 위력 압도 → 노바를 핵심 화력으로 삼는 과부하 누커(8.1 참조).
- **프록시:** 브레이크가 보호막 충전이라 과열 게이지가 차지 않음 → 노바를 거의 못 쓰는 대신 단단함을 택한 트레이드오프.

## 8. 기체(캐릭터) — 물리 파라미터 필터 설계

**대원칙:** 코어 로직은 절대 캐릭터별로 다시 코딩하지 않는다. 기체는 코어의 **물리 파라미터(개수/속도/크기/생존시간/격발 트리거 등)**만 왜곡하는 "하드웨어 필터"다. 이로써 신규 캐릭터 추가가 변수 몇 줄로 끝나고, 개발자가 의도치 않은 사기 조합이 창발적으로 터진다.

> 📝 표의 "압축 시" 효과(레거시 +50%, 광랜 복리 등)는 모두 **인접 자동 압축(3.3)이 일어나는 순간**을 가리킨다(겹쳐 압축 폐기 후에도 개념 유효).

| 기체 | 물리 파라미터 필터 | 성장 성향 | 정체성 |
|---|---|---|---|
| **레거시-99** (기본) | 표준 + 압축 시 동일 계열 등장 확률 +50% | 안정적·빠른 성장 | 빌드 안정성 (정석 빌드업) |
| **프록시** (탱커) | 투사체 속도 −80%, 크기 +300%, 지속 +500% / 2칸 고정 무적 장갑(모든 일반 코어의 조커) / 브레이크 시 체력 소모 대신 **초당 +1% 보호막 충전** | 점진적 대기만성 | 기하학적 트랩존(결계) |
| **광랜-X** (스피드) | 탄환 개수 ×3, 대미지 ×0.3 / 압축 성공 시 사격속도 영구 복리 스택 | 안정 시작·후반 기하급수 | 초고속 물량 탄막 |
| **글리치-Zero** (하이리스크) | 코어가 사격 중 자체 과열 → 일시 정지(쿨다운) 반복으로 **지속 화력 불안정(약함)**, 그 과열이 **노바 게이지로 전환**(평소 약함 = 노바 연료) / 브레이크 드레인 스택 감소(최대 0) / **노바 위력 압도적**(정예·보스에도 큰 피해) / 보스 페이즈 전환 순간 노바 = 저격 폭딜 | 평소 약함·순간 폭발 | 과부하 누커 (8.1 상세) |
| **딥웹** (도박, *원칙 예외*) | 인접 압축이 레시피 무시 **무작위 룰렛**(대박/중간/꽝) / 코어삭제 재굴림 / 저주 시 슬롯 일시 잠금 / 극악 확률 전용 코어 **제로데이** | 변칙·도박 | 와일드 압축 잭팟 (8.2 상세) |

> **모든 코어는 모든 기체가 100% 동일하게 획득·조합**할 수 있다. 차이는 "어떻게 해석/증폭하느냐"뿐이다. 동일한 6전기 도배도 광랜=탄막, 프록시=전기 격벽, 글리치=과부하 사출로 완전히 달라진다.
>
> **밸런스 철학:** 하한선(쓰레기 조합)은 막되, 상한선(유저가 발견한 폭딜 조합)은 천장을 뚫어둔다 — "의도된 불균형"이 곧 재미다.

### 8.1 글리치-Zero 상세 — 과부하 누커
광랜(지속 우상향 딜러)의 정반대 결: **평소 약하고 결정적 순간에 폭발**하는 하이리스크 하이리턴 누커. 슬롯 칸 수는 건드리지 않아(슬롯 성장의 보람 유지) 리스크를 "불안정 사격"이라는 새 축으로 건다.

- **불안정 과열 사격 (리스크):** 슬롯 6칸을 정상 사용하되, 코어는 사격할수록 자체 과열되어 일정 시간 연사하면 **일시 정지(쿨다운) 후 재가동**을 반복한다 → 지속 화력이 들쭉날쭉, 평상시 DPS가 낮다. ("글리치=시스템 결함" 컨셉/비주얼과 일치: 코어가 지직거리다 멈춤.)
- **과열 → 노바 연료 (전환):** 이 사격 과열이 **노바 게이지로 전환**된다 → 불안정하게 끊기는 만큼 노바가 빨리 찬다. "평상시의 약함"이 곧 "노바의 연료".
- **브레이크 자유:** 브레이크 드레인이 스택 감소(최대 0) → 위험 없이 오래 브레이크(방어 격자) 유지 가능 → 과열·노바 충전을 적극적으로 돌린다.
- **노바 위력 압도 (리턴):** 다른 기체 노바가 "잡몹 정화" 수준이라면, 글리치 노바는 **정예·보스에게도 큰 피해**를 주는 폭발. 평소 낮은 화력을 노바 폭발로 메우는 루프.
- **보스 페이즈 저격 (시그니처):** 보스의 **페이즈 전환 순간**에 정확히 노바를 터뜨리면 추가 효과(대폭딜 또는 경직 연장) 발동 → 기존 CRITICAL FORMAT의 저격 쾌감을 노바 타이밍으로 계승.
- **루프:** 불안정 사격으로 버티며 과열 충전 → 브레이크로 방어하며 게이지 채움 → 임계에서 노바 폭발로 순간 고점 → 보스전엔 페이즈 전환 타이밍 저격. *(위성 포탑은 폐기.)*

### 8.2 딥웹 상세 — 와일드 압축 잭팟
도달은 어렵지만 고점이 압도적인 도박 기체. 다른 기체가 결정론적으로 압축을 "설계"한다면, 딥웹은 결과를 예측할 수 없는 룰렛을 "굴린다". 너프 덩어리가 아니라 **양방향 변동성**이 정체성.

- **무작위 룰렛 압축:** 인접 압축이 색·레시피를 무시하고 결과가 도박으로 결정된다.
  - **대박 (~20%):** 다른 기체가 못 만드는 한 단계 위 결과 — 즉석 티어3급, 또는 무작위 Affix가 붙은 변이 코어(티어3+Affix 풀 재활용, 신규 자산 0). 딥웹의 천장.
  - **중간 (~50%):** 평범한 동급 결과.
  - **꽝/저주 (~30%):** 마이너스 Affix 코어(디도스=시야 일부 가림, 랜섬웨어=사격 시 재화 소량 소모 등 — 기존 Affix의 음수 버전). **빌드를 망치진 않지만(사격은 됨) 거슬려 재굴림하고 싶은 수준.**
- **저주 슬롯 일시 잠금:** 꽝 저주 코어는 **교체(재굴림)하기 전까지 그 슬롯이 일시 잠금**된다. 그냥 두면 거슬리고, 고치려면 코어를 써야 하는 압박 → 재굴림을 유도. *(체력 영구 잠금은 삭제.)*
- **재굴림 = 코어 삭제:** 저주 슬롯에 새 코어를 덮어쓰면(3.3 덮어쓰기 흡수) 기존 코어가 삭제되고 룰렛이 다시 돈다. **비용은 "어렵게 모은 코어를 통째로 버리는 것" 자체** → 코어가 한정적이라 "지금 손절할까, 한 번 더 굴릴까"의 긴장. 무한 재굴림 자연 억제.
- **설계 불가 = 정체성:** 결과를 못 정하니 링크 배치학·의도적 색조합 빌드를 거의 못 쓴다. 단점이 아니라 "다른 기체는 설계하고, 딥웹은 굴린다"는 정체성.

#### 제로데이 (Zero-Day) — 딥웹 전용 극악 확률 유니크 코어
티어 체계 밖의 단일 유니크 코어. "알려지지 않은 치명적 취약점" 컨셉 — 시스템을 통째로 깨부수는 잭팟.

- **등장:** 딥웹 룰렛 **대박 안에서도 극악 확률**로, **3섹터(21~30방)에서만** 출현. 별도 해금 불필요(딥웹 자체가 첫 클리어 후 해금이므로, 볼 자격은 자연히 클리어 경험자로 제한됨).
- **효과:** 획득 즉시 **압도적 연출과 함께 최종 보스로 직행 → 순식간에 처형 → 클리어.** 말 그대로 잭팟.
- **사건화:** 일반 클리어와 구분되는 전용 도감/업적("제로데이로 커널을 해킹했다") + 전용 글리치 연출·사운드. 확률은 극악이어도 "터졌을 때의 사건"으로 박제.
- **딥웹 전용:** 다른 기체는 무슨 수를 써도 못 본다 → 딥웹을 키울 동기.

#### 딥웹 해금
- **선행 조건:** 게임 정식 클리어 1회 이상(가장 변칙적인 기체 → 게임을 한 번 이해한 뒤 개방).
- 조건 충족 후 상점에 등장, **재화로 구매**(9.1 LV3 캐릭터 계열 + 추가 선행 조건).

> ⚠️ **밸런싱 검증 대상:** 대박 확률 20%가 압축 1회당이면 후반 누적으로 과강해질 수 있음. 제로데이 극악 확률의 구체값. 저주 강도. → 밸런싱 패스(15장)에서 확정.

## 9. 아웃게임 성장 — 통합 커널 마스터리

캐릭터별 스킬트리는 두지 않는다(새 캐릭터가 약하게 느껴지는 박탈감 방지). **계정 전체가 공유하는 단일 스킬트리** 하나만 운영하고, 새 캐릭터는 해금 즉시 100% 완성된 고유 패시브로 투입된다.

### 9.1 4단계 보안등급 해금 로드맵
- **LV1 USER — 태그 시스템:** 첫 판부터 기본 제공. 속성 강화 노드.
- **LV2 ROOT — 소켓 & 링크 배치학:** 누적 깊이 달성/보스 처치로 해금.
- **LV3 DEV — 캐릭터:** 인게임 재화로 구매(스킬트리 없이 완성형 패시브 즉시). *단 딥웹은 정식 클리어 1회 선행(8.2).*
- **LV4 ADMIN — Affix 모듈:** 하드코어 업적 클리어로 해금 → 필드에 변이 코어 드롭 시작.

### 9.2 링크(배치학) 3종
- **인접 링크:** 양옆 슬롯 같은 태그 → 지속 버프.
- **대칭 링크:** 180도 마주보는 슬롯 같은 태그 → 중앙 관통 에너지 격벽.
- **풀하우스:** 6칸 모두 채움 → 캐릭터 궁극 필터 가동.

### 9.3 사망 / 클리어 정산 (환전 루프)
- **판 종료:** 기체 체력 0 → 그 자리에서 종료. 클리어(30방 격파)도 종료. 두 경우 모두 **정산 화면**으로 이동.
- **인게임 코어·빌드는 전부 소멸**(로그라이트 기본 — 매 판 새로 빌드업). 영구히 남는 것은 **정산 재화**뿐.
- **재화 = 단일 통화.** 도달 깊이에 비례하되 구간을 끊어 초반도 의미 있게:
  - 매 웨이브 도달마다 소량 누적.
  - 섹터 경계(10·20·30방) 돌파 시 **큰 보너스 덩어리** → "10방만 가도 한 단계 보상".
  - **첫 도달 보너스(First Clear):** 그 웨이브에 처음 도달했을 때만 주는 일회성 큰 보너스 → 신기록 갱신 = 재화 점프.
- **성취 박제:** 정산 화면에 완성한 티어3 개수·발견한 색조합을 도감처럼 기록(코어는 사라져도 성취감은 남김 — 업적 연계).

### 9.4 영구 강화 — 단일 통화 + 병렬 3갈래
단일 통화를 한 곳에 몰빵하지 못하도록, **슬롯에 진척 조건(잠금)을 부가**해 초과분이 자연히 다른 축으로 흐르게 한다.

- **① 슬롯 개방 (대형 마일스톤, 3단계):** 3칸 시작 → 6칸. **재화 + 진척 조건** 둘 다 충족해야 개방.
  - **4번째:** 매우 저렴 + (한 번이라도 섹터1 보스 도달) → 첫 사망 후 1~2판 내 거의 확정.
  - **5번째:** 중간 + (섹터2 보스 도달).
  - **6번째:** 비쌈 + (섹터3 진입) → "최종 섹터 도전 자격증".
  - → 돈만으로 못 열리니, 진척이 막힌 동안 재화는 아래 ②③으로 흐른다(자연 병렬).
- **② 기체 피지컬 (소형 점증) — 슬롯과 상호 우회 불가능한 이중 게이트:**
  - **회전 속도:** 극초반 1~3단계만 증속 후 **상한 고정**(빠를수록 조준 어려워지는 역설 회피). 기본은 약간 답답하게 → 첫 성장 동기. = **섹터1 진입 게이트**(호 이동 추적).
  - **최대 체력:** 점증. = **피탄 허용/실수 마진 게이트**(섹터2~3 정렬 실패 시 생존). *피탄 전제가 아니라 안전 마진 — 완벽 운용 시 노 히트 가능.*
  - **요격 처리량(관통/광역):** 점증. 브레이크 사격이 한 번에 꿰뚫는 적 수·상쇄 범위. = **물량 처리 게이트**(섹터3 고밀도 호). 조준과 무관한 순수 시스템 강화.
  - **브레이크 효율(드레인 감소) + 노바 충전 효율:** 점증. = **지속 방어·전역 정화 게이트**(섹터3 장기전·완전 포위).
  - *정밀도(브레이크 조준)는 순수 유저 스킬 영역 → 시스템이 강화하지 않음(액션성 보존).*
- **③ 시작 부스트 (편의):** 시작 코어 +1(무작위 저티어), 정비 궤도 회전 감속 등.

> **운 클리어 3중 봉쇄:** ① 슬롯 병목(화력 물리 상한) ② 적 곡선을 "슬롯·피지컬 요구치 미만은 수학적으로 처리 불가"로 설계 ③ 피지컬 DPS 직접 강화 없음(게이트 우회 차단). 단 **노 히트 가능성은 항상 보장** — 어떤 웨이브도 "완벽 운용해도 무조건 피탄"인 구성을 두지 않는다.

### 9.5 이어하기 (Continue)
- **판당 1회.** 죽은 자리에서 체력 일부 회복 + 빌드 유지한 채 재개.
- 발동: 광고 시청 또는 유료 재화.
- **보스전 한정:** 중간 보스(10·20방)·최종 보스(30방) 전투 중 사망에만 사용 가능. 일반 웨이브 사망에는 적용되지 않는다(초반 남발 시 성장 압박이 풀려 아웃게임 루프가 약해지는 것 방지 + "여기까지 왔는데 보스에서 아깝게 죽었다"는 가장 부활 욕구가 큰 순간에 집중).

## 10. 적(Malware) 디자인

이동 불가한 중앙 기체를 향한 **방향성 압박**과 **사격선 방해**가 전부. 잡몹은 단일 패턴(가벼운 연산), 정예만 복잡한 기믹.

### 10.1 일반 잡몹 (6종)
- ⚡ **클럭 러너:** 중앙으로 최단거리 직진. 에임 테스트.
- ⚡ **핑 바운서:** 외곽 일정 궤도 공전. 멀리 쏠 때 총알 대신 맞는 탄막 차단벽.
- 🔥 **비트 슈터:** 제자리에서 중앙으로 단발 저격. 조준 강제 전환 유발.
- 🔥 **스팸 마인:** 고정 스폰. 기체와 자신 사이에 붉은 경고 레이저 → 그 선을 쏘면 폭발(사격 각도 봉쇄).
- 🕳️ **루프 플라이어:** S자 궤적으로 비틀거리며 접근(예측 사격 교란).
- ⚙️ **정크 실더:** 고체력·무공격, 느리게 전진. 뒤따르는 잡몹 호위 방벽.

### 10.2 특수 정예 (맵당 1~2기)
- 🕳️ **위상 왜곡자(퍼플):** 자력선을 꽂아 **회전 방향을 반대로 꼬아버림.** ※ 2안에서는 자동회전과 탭 반전 모두를 뒤집어 더 위협적 → 최우선 점사.
- ⚙️ **하드웨어 크러셔(그레이):** 거대 실드. 중앙 도달 시 6칸 슬롯 하나를 **잠금(Lock).** 잠긴 슬롯은 회전은 같이 하되 사격·압축·링크 시너지에서 제외되고 회색 글리치 우리(cage)가 씌워진다. **코어 데이터는 보존**되며, 크러셔를 **처치하는 즉시 잠금 해제**된다.
  - **카운터플레이:** 내리찍기 직전 약 0.7초간 망치를 들어 올리는 **예고 모션** → 이 사이에 처치하면 잠금 자체를 막는다.
  - 전면 실드+고체력이라 정면 사격은 잘 안 박힌다 → 측면 조준 전환 또는 장갑 파괴(레드×그레이 = 열역학 파쇄)로 뚫어야 한다(빌드 연계).
  - 정예는 맵당 1~2기 → 동시 잠금 최대 2칸, 생존 가능 범위.

### 10.3 공습 모델 — 연속 각도 분포 (이산 "방향 수" 폐기)
적은 정해진 N방향이 아니라 **360도 원주 어디서든** 등장한다(스피디한 게임에 맞는 자연스러운 설계). 위협은 정수가 아니라 **연속 각도량**으로 기술한다.

- **위협 각도 폭(spread):** 적이 분포하는 원주 범위. 협호(좁게 몰림)면 브레이크 집중이 잘 통하고, 광호·전방위면 회전 스윕/잦은 전환 필요.
- **최대 밀도(peak density):** 가장 몰린 호의 적 집중도. 높을수록 고화력 코어를 정렬해도 벅참.
- **위협 호 이동:** 위험 호가 원주를 옮겨다니는 속도. 탁(회전 반전)으로 추적.

**화력 = 코어 조준 각도 ∩ 적 각도(겹침).**
- **자동 회전:** 6조준선이 원주를 스윕 → 넓고 얕게(전방위에 유리, 순간 화력 약함).
- **브레이크:** 6코어가 고정 6각도에 지속 사격 → 좁고 깊게(몰린 호에 집중, 빈 각도엔 0).
- **위상 정렬(핵심 깊이):** 6코어는 화력이 불균등(티어3이 대부분). 브레이크의 진짜 의미는 **고화력 코어(티어3)가 위험 호를 향하는 순간에 멈추는 것**. 탁으로 회전을 튕겨 고화력 코어를 위험 호로 굴린 뒤 꾹. 정렬 성공=폭딜, 실패=보조 코어 화력만. 위협 호 지속시간 안에 정렬 기회가 최소 1회 와야 노 히트 성립.

> **등장 예고:** 360도 자유 등장의 가독성을 위해, 스폰 직전 외곽 해당 각도에 **짧은 네온 경고 글로우**가 번쩍인다(텍스트 제로 UI 부합 + 노 히트 가능성의 정보 토대).

### 10.4 적별 위협 비용 (Threat Cost)
스폰은 고정 테이블이 아니라 **예산 기반 랜덤 조합**으로 굴린다. 각 적에 위협 비용을 매겨, 웨이브 예산 한도 안에서 무작위 조합(클럭 러너=2 기준).

| 적 | 위협 비용 | 근거 |
|---|---|---|
| 클럭 러너 | 2 | TTK 1.0, 직진(기준) |
| 핑 바운서 | 2 | TTK 1.0, 공전 |
| 루프 플라이어 | 3 | TTK 1.2, S자 |
| 비트 슈터 | 3 | TTK 1.5, 원거리 저격 |
| 스팸 마인 | 4 | 사격 각도 봉쇄 |
| 정크 실더 | 5 | TTK 2.5~3.0, 고체력 방벽 |
| 위상 왜곡자(정예) | 8 | 회전 반전(정렬 교란) |
| 하드웨어 크러셔(정예) | 8 | 슬롯 잠금 + 고체력 실드 |

**예산 기반 랜덤 스폰 + 3제약:** 총 위협량(예산)은 곡선대로 고정, 구성은 매 판 무작위 → 패턴 암기 방지. 제약: ① **각도 분산**(예산이 한 호에 몰빵 금지, spread 상한 준수) ② **적 해금 게이트**(각 적의 등장 가능 웨이브 고정 — 학습 순서·정예 조기등장 방지) ③ **연속 동일 조합 억제**(직전 웨이브와 유사 구성 회피).
*보스(10·20·30)는 패턴 공략이 재미 → 고정 패턴 유지, 랜덤 미적용.*

### 10.5 기체 디자인
구형 또는 정삼각형의 미니멀 아키텍처. 중심에 백색 순수 에너지원, 외곽에 장착 코어 색상을 반영하는 네온 프리즘, 반투명 홀로그램 6궤도 링이 공전.

## 11. 레벨 구조 — 3섹터 × 10웨이브 = 30방

반복 도전(아웃게임 성장)으로 점차 깊이 진입하며, 최종엔 **2~3개 티어3 + 보조 저티어**가 동시에 화면을 수놓는 화력쇼를 제공.

- **인게임 1세션:** 약 15~20분.
- **아웃게임 수명:** 반복 20~30회 / 10~15시간+.

### 11.1 보스 구성
| 섹터 | 구간 | 중간/최종 보스 | 기믹 | 클리어 보상 |
|---|---|---|---|---|
| 1 커널 진입 | 1~10 | **WALL_E(방화벽 게이트)** | 3개 회전 방패로 사격 방어 | **슈퍼 정비 궤도**(3.7): 후보 5~6개 중 2개 확보, 회전 감속 |
| 2 데이터 멜트다운 | 11~20 | **ANTI_V(오염된 백신)** | 조준선 위 에러 블록 생성 + 포탑 다수 스폰 | **슈퍼 정비 궤도**(3.7) |
| 3 중앙 프로세서 | 21~30 | **KERNEL_PANIC(신경망 붕괴)** | 화면 테두리 전체가 보스 HP. 4속성 무제한 물량 | God-Mode 화력쇼로 클리어 |

### 11.2 적 강도 곡선 (3섹터 × 10웨이브)
**측정 단위:** TTK(Time To Kill, 기본 무기로 적 1기 처치 시간 — 클럭 러너=1.0초 기준) / TTR(Time To Reach, 스폰→중앙 도달 시간 = 요격 제한시간, 기본 4.0초). 적 체력은 TTK 배수로 표기.

**섹터별 주력 레버:** 섹터1=위협 각도 폭(spread) / 섹터2=위협 호 이동 속도 / 섹터3=밀도+복합. 세 레버를 분리해 섹터마다 위협의 "결"이 다르게.

#### 섹터 1 — 커널 진입 (1~10) · 레버: spread / 위상 정렬 학습
요구 성장: 슬롯 3칸 + 회전 속도 1~3단계(초반 게이트). 정렬 실패가 치명적이지 않은 학습 구간.

| W | 예산 | 위협 각도 폭 | 밀도 | 호 이동 | 정렬 요구 | TTR |
|---|---|---|---|---|---|---|
| 1 | 4 | 단일 협호(~30°) | 낮 | 정지 | 불필요 | 4.0 |
| 2 | 6 | 협호 1~2 | 낮 | 느림 | 기본 | 4.0 |

#### 밸런스 원칙 (전 웨이브 검산)
- **하한선(운 클리어 봉쇄):** 요구 슬롯·피지컬 미만은 화력/커버리지 부족으로 **수학적 확정 사망**(운 개입 0). 섹터3은 티어3 2~3개+노바가 수학적 필수.
- **상한선(노 히트 보장):** 가장 가혹한 웨이브(29)도 슬롯6+티어3 3개+전 피지컬 상위+완벽 운용 시 모든 호 제때 커버 가능하도록, 밀도를 **처리 한계 바로 아래**에 설정. "완벽해도 무조건 피탄"인 구성은 없음. 마진은 섹터가 깊을수록 0에 수렴.
- **방향당 화력 검산(코어 DPS 확정 후):** 방향당 위협(호 적 체력합÷TTR) ≤ 그 호에 정렬된 코어 화력. 위상 정렬·요격 처리량 게이트가 여기서 작동. → 6대 스탯 베이스라인 확정 시 정밀 검증(15장).

### 11.3 보상 공급 밸런스
- 매 웨이브 보상은 **3지선다**(리롤 없음, 가중 추첨 3규칙 — 3.6). 슬롯 제한·에너지 소모 패널티로 진화 속도를 제어.
- 섹터별 A(코어)/B(스탯) 비율은 3.5 참조. 섹터1 빌드업 위주, 섹터3 스탯 펌핑 위주.
- 중간 보스 보상은 **슈퍼 정비 궤도**(3.7) — 확정 지급이 아니라 풍부한 선택지로 빌드업 가속.

### 11.4 보스 추락 일관성
보스는 **동속 추락형 동반체**로 등장(상대 Z축 속도 0). 가속/감속은 Z축 이탈이 아니라 **X-Y 평면상 중앙 수축(가속) / 외곽 후퇴(감속)**으로 표현해 6방향 수평 사격 평면을 유지한다. 카메라·조작 프레임은 절대 바뀌지 않는다.

### 11.5 슬롯-섹터 진행 곡선 (아웃게임 연동)
슬롯 수가 곧 빌드 상한이므로, 슬롯 개방(9.4 ①)이 도달 가능 깊이를 자연스럽게 결정한다. 막연한 벽이 아니라 **"슬롯 N칸 → 섹터 N 도전 자격"**이라는 눈에 보이는 목표.

| 슬롯 | 대략적 도달 한계 | 빌드 수준 |
|---|---|---|
| 3칸 (시작) | 섹터1 중간보스(10방) 언저리 | 티어2 1~2개 |
| 4칸 | 섹터1 클리어 ~ 섹터2 진입 | 첫 티어3 안정 운용 |
| 5칸 | 섹터2 돌파 | 티어3 2개 빌드 |
| 6칸 | 섹터3 ~ 최종 보스 도전 | 티어3 2~3개 + 보조 풀빌드 |

- 슬롯 개방엔 진척 조건이 걸려 있어(9.4), 다음 슬롯을 열려면 더 깊이 가봐야 한다 → 그라인드가 아니라 **실력 진척**으로 성장.
- "돈은 충분한데 5번 슬롯이 안 열린다 → 이번 판 목표는 섹터2 도달" 식의 명확한 단기 목표가 매 판 생긴다.
- 슬롯을 모으는 동안에도 피지컬(9.4 ②)이 매 판 올라 "전진감"을 유지 → 질림 방지.

## 12. 시각적 UI 피드백

- **아군 HP:** 기체 네온 라인의 글리치 강도 + 화면 테두리 붉은 에러 비네팅 + 위험 시 이명·붉은 파편 탈락.
- **적 HP:** 장갑 파편화 → 내부 붉은 에러 코어 노출·진동.
- **보스 HP:** 화면 테두리 전체를 감싸는 **거대 네온 방화벽 서클**. 타격 시 덩어리째 썰려 나감.
- **대미지:** 숫자 폰트 ❌. 속성 색 픽셀 스파크 팽창. 강력한 적중 시 0.05~0.1초 **Hit-Stop(역체감)** + 사각형 글리치 충격파.

## 13. 최적화 & 기술 아키텍처

보급형 스마트폰에서 발열 없이 60fps 유지가 강박적 기준.

- **객체 풀링(Object Pooling):** 스테이지 진입 전 최대치 탄환/적을 선할당. 런타임 Instantiate/Destroy 전면 금지 → GC 렉 차단.
- **드로우 콜 배칭 & 인스턴싱:** 텍스트 UI 없는 이점을 살려 4속성별 단일 머티리얼 + 가벼운 Unlit 셰이더. GPU Instancing으로 동일 색 탄환 수백 개를 1드로우콜로 렌더.
- **이벤트 기반 연산:** 스탯 합산/특수효과는 매 프레임이 아니라 **코어 충돌 그 프레임에만** 델리게이트로 단발 처리.
- **VFX 다이어트(Sprite Baking):** 티어3 폭발·오버히트 노바는 3D 파티클 대신 프레임 단위로 구운 2D 스프라이트 시트로 재생.
